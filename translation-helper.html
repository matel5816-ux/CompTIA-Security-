📝 翻譯工作流程

第1步：找到未翻譯的題目

在 index.html 中查找沒有 zh: 字段的題目

第2步：複製題目資訊

複製英文題目、選項和正確答案

第3步：翻譯和分析

翻譯題目和選項，撰寫詳細的觀念解析

第4步：格式化並插入

按照規定格式加入 zh 字段到 index.html

💡 翻譯格式範例

zh: {
  "q": "中文題目翻譯",
  "opts": [
    "選項A中文翻譯",
    "選項B中文翻譯",
    "選項C中文翻譯",
    "選項D中文翻譯"
  ],
  "explain": "詳細的觀念解析："
}

⚠️ 重要提示

  • ✓ 每個選項翻譯應簡潔清晰
  • ✓ 解析應解釋正確答案的原因和其他選項為何錯誤
  • ✓ 使用繁體中文
  • ✓ 保持與已翻譯題目的風格一致
  • ✓ 逐次提交 git commit，不要一次性提交所有改動

🎯 建議做法

  1. 每天翻譯 10-20 道題目
  2. 每次翻譯完成後立即 commit
  3. 先翻譯前 100 道題目（最常見和最重要的）
  4. 建立良好的翻譯和解析品質標準
  5. 考慮使用 AI 輔助初稿，然後手工審核和改進
